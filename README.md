# Gabby Sobieski — Portfolio

A single-page portfolio site (static HTML/CSS/JS, no build step). Ready for GitHub Pages.

## Files
- `index.html` — the whole site (styles and scripts are embedded)
- `images/` — drop your photos here (see `images/PHOTO-GUIDE.txt` for exact filenames)

## Preview locally
Just double-click `index.html` to open it in a browser. The YouTube embed and Google Fonts need internet.

## Publish free on GitHub Pages
1. Create a new repository on GitHub (e.g. `portfolio`). Make it **Public**.
2. Upload `index.html`, the `images/` folder, and this `README.md` (drag-and-drop in the GitHub web UI works fine).
3. In the repo, go to **Settings → Pages**.
4. Under **Build and deployment → Source**, choose **Deploy from a branch**.
5. Set branch to **main** and folder to **/ (root)**, then **Save**.
6. Wait ~1 minute. Your site goes live at `https://<your-username>.github.io/<repo-name>/`.

### Optional: cleaner URL
- Naming the repo `<your-username>.github.io` publishes it at `https://<your-username>.github.io/` (no subfolder).
- Or add a custom domain under Settings → Pages if you own one.

## Updating later
Replace a photo by uploading a file with the same name into `images/`, or edit text directly in `index.html`. Changes go live automatically a minute after you commit.
